# 🤝 Contributing to "Вікторина (Python Tkinter)"

Дякуємо, що хочете зробити внесок у проєкт! 🎉

Цей документ містить правила та рекомендації для участі в розробці застосунку "Вікторина".

---

## 📌 Code of Conduct

Беручи участь у проєкті, ви погоджуєтесь дотримуватись загальних правил ввічливої та професійної взаємодії.

Будь-яка неприпустима поведінка може бути причиною відмови у прийнятті змін.

---

## 📂 Issues

### Як використовуються Issues:
- повідомлення про помилки
- пропозиції нових функцій
- загальні питання щодо проєкту

### Рекомендації:
- описуйте проблему чітко
- додавайте скріншоти (якщо можливо)
- вказуйте кроки для відтворення помилки

---

## 🛠 Як зробити внесок

### 🔹 Fork репозиторію
Створіть копію проєкту у своєму GitHub акаунті.

---

### 🔹 Клонування

```bash
git clone https://github.com/Arzenh/QAprKholoshevskyi.git
cd QAprKholoshevskyi
```
### 🔹 Створення гілки

```bash
git checkout -b feature/new-feature
```
## Типи гілок:

feature/ — новий функціонал
fix/ — виправлення помилок
refactor/ — покращення коду
docs/ — документація

## 💻 Внесення змін

```bash
git add .
git commit -m "feat: опис змін"
```

## Типи комітів:

feat: нова функція
fix: виправлення помилки
docs: документація
style: форматування
refactor: зміни структури коду
test: тести
🚀 Відправка змін
git push origin feature/new-feature

Після цього створіть Pull Request у GitHub.

## 🔄 Pull Requests

Етапи:
Створити Pull Request
Описати внесені зміни
Дочекатися перевірки
Внести правки (якщо потрібно)
Після схвалення зміни будуть прийняті
🧪 Тестування

### Перед створенням Pull Request переконайтесь:

програма запускається без помилок
інтерфейс працює коректно
відповіді у вікторині рахуються правильно
фінальний результат відображається

## 🧱 Стиль коду

чистий і читабельний код
зрозумілі назви змінних
логічна структура програми
мінімум зайвих бібліотек
дотримання базового PEP8

## ⚠️ Відомі обмеження

Якщо знайдені проблеми — вони фіксуються в Issues.

## 📚 Додаткова інформація

Проєкт створений на Python з використанням бібліотеки Tkinter.

## ❤️ Дякуємо!

Будь-який внесок, навіть невеликий, допомагає покращити проєкт.
